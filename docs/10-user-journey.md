# 10 - User Journey

## What is a User Journey?
A user journey maps the steps a user takes to accomplish a goal using the system — from their initial need to task completion.

---

## Journey 1: Customer Books an Appointment

**Actor:** Tanvir (Customer Persona)
**Goal:** Book a haircut appointment online

| Step | Action                                 | System Response                                      | Emotion     |
|------|----------------------------------------|------------------------------------------------------|-------------|
| 1    | Visits the BookEase website            | Landing page loads with a "Book Now" button          | Neutral     |
| 2    | Clicks "Book Now" or "Sign Up"         | Registration form appears                            | Curious     |
| 3    | Creates an account with email/password | Account created, redirected to service listing       | Engaged     |
| 4    | Views list of available services       | Services shown with name, duration, and price        | Interested  |
| 5    | Selects "Haircut" service              | Calendar with available dates appears                | Focused     |
| 6    | Selects a date and available time slot | Booking summary shown (service, date, time)          | Excited     |
| 7    | Confirms the booking                   | Success message + confirmation email sent            | Satisfied   |
| 8    | Views "My Bookings" page               | Booking appears with status "Confirmed"              | Confident   |

---

## Journey 2: Admin Manages the Day's Schedule

**Actor:** Farida (Admin Persona)
**Goal:** View today's appointments and cancel a no-show

| Step | Action                                  | System Response                                          | Emotion     |
|------|-----------------------------------------|----------------------------------------------------------|-------------|
| 1    | Goes to BookEase and logs in as Admin   | Admin dashboard loads                                    | Neutral     |
| 2    | Views today's booking overview          | List of today's appointments with customer names & times | Focused     |
| 3    | Sees a booking from a known no-show     | Booking details visible with a "Cancel" option           | Frustrated  |
| 4    | Clicks "Cancel" on the booking          | Confirmation prompt appears                              | Cautious    |
| 5    | Confirms cancellation                   | Booking removed from schedule, customer notified         | Relieved    |
| 6    | Adds a new available slot to fill the gap | Slot becomes visible for customers to book             | In control  |

---

## Journey 3: Customer Cancels a Booking

**Actor:** Tanvir (Customer Persona)
**Goal:** Cancel a booking he can no longer attend

| Step | Action                          | System Response                                 | Emotion  |
|------|---------------------------------|-------------------------------------------------|----------|
| 1    | Logs in to BookEase             | Dashboard shows upcoming bookings               | Neutral  |
| 2    | Clicks on his upcoming booking  | Booking detail view with "Cancel" button        | Hesitant |
| 3    | Clicks "Cancel Booking"         | Confirmation prompt: "Are you sure?"            | Careful  |
| 4    | Confirms cancellation           | Booking removed, confirmation email sent        | Relieved |
