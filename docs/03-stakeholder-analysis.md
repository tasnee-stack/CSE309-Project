# 03 - Stakeholder Analysis

## Overview
A stakeholder is any person or group who is affected by or has an interest in the BookEase system. Understanding stakeholders helps ensure the system is designed to meet the right needs.

---

## Stakeholder Table

| Stakeholder         | Role                   | Interest / Expectation                                                  | Influence Level |
|---------------------|------------------------|-------------------------------------------------------------------------|-----------------|
| Customers / End Users | Book appointments      | Easy to use, fast booking process, confirmation emails, ability to cancel | High            |
| Business Owner / Admin | Manage schedule & services | View/manage bookings, set availability, manage services, see analytics | High            |
| Developer (You)     | Build and maintain app | Clear requirements, manageable scope, working system                    | High            |
| Course Instructor   | Evaluate the project   | Proper documentation, working code, GitHub project management           | Medium          |

---

## Detailed Stakeholder Profiles

### 1. Customer (End User)
- **Who they are:** Individuals who want to book a service (e.g., a haircut, doctor visit, tutoring session)
- **Goals:** Find available slots quickly and confirm bookings without calling
- **Pain Points:** No visibility into real-time availability; hard to cancel or reschedule
- **Expectations from BookEase:** Clean UI, real-time availability, email confirmation, easy cancellation

### 2. Business Owner / Admin
- **Who they are:** The owner or manager of the service business
- **Goals:** Reduce manual scheduling work, prevent double-bookings, grow bookings
- **Pain Points:** Missed calls, no-shows, scattered records, manual reminders
- **Expectations from BookEase:** Dashboard to manage services & slots, view upcoming appointments, cancel/confirm bookings

### 3. Developer
- **Who they are:** The solo developer building this project (you)
- **Goals:** Deliver a functional, documented, well-structured app
- **Expectations:** Achievable scope with React + Node.js stack

### 4. Course Instructor
- **Who they are:** The evaluator assessing the project
- **Goals:** See proper software engineering practices applied
- **Expectations:** Complete GitHub project board, documentation, pull requests, working app
